Описание

Компоненты позволяют выделять большие фрагменты шаблонов в переиспользуемые модули, а также прикреплять к ним необходимые методы (методы ближайшего компонента просматриваются в первую очередь, когда Autosync встречает пользовательскую функцию в условии или обработчике события). В настоящий момент компонент - это обычный объект со свойствами template и funcs, который добавляется в объект as.components по ключу:


as.components['items-list'] = {
   template: '<ul asmodel="data.items">\
      <li data-id="{$value[\'id\']}" as-onclick="handleClick(event)">{$value[\'title\']}</li>\
   </ul>',
   funcs: { handleClick: function(event) { console.log('Click!') } }
}

Далее вы можете использовать компонент в любом шаблоне как кастомный элемент:


<div asmodel="data.tasks">
   /* ... */
   <items-list></items-list>
   /* ... */
</div>
Рекурсия

Вы можете ссылаться в шаблоне компонента на сам компонент, если того требует логика вашего приложения. Будьте осторожны, чтобы не получить при этом бесконечную рекурсию.

Например, идеальным вариантом будет вызов компонента внутри шаблона, выводящего коллекцию: в реальных данных у вас не будет бесконечной вложенности, и в какой-то момент рекурсия завершится при нулевом размере списка.

Чтобы рекусия работала, вам понадобится использовать относительные пути к объектам. Взгляните на данный пример:


as.components['items-list'] = {
   template: '<ul asmodel=".items">\
      <li">{$value[\'title\']}</li>\
   </ul>'
}

var data = { items:
   [
      { title: 'Food', items:
         [
            { title: 'Apples', items: [] },
            { title: 'Cookies', items: [] },
            { title: 'Fish', items:
               [
                  { title: 'Salmon', items: [] },
                  { title: 'Herring', items: [] }
               ]
            },
            { title: 'Water', items: [] }
         ]
      },
      { title: 'Other', items:
         [
            { title: 'Wash powder', items: [] },
            { title: 'Paper towels', items: [] }
         ]
      }
   ]
}
               

<h1>Shopping list</h1>       
<ul asmodel="data.items">
   <li>
      <div>{$value['title']}</div>
      <items-list></items-list>
   </li>
</ul>

Обратите внимание на то, как немного переписав модель и шаблон, можно сделать заголовок списка покупок корневым элементом структуры:


var data = { items:
   [
      { title: 'Shopping list', items:
         [
            { title: 'Food',  items: [ /* ... */ ] },
            { title: 'Other', items: [ /* ... */ ] }
         ]
      }
   ]
}
               
   
<ul asmodel="data.items">
   <li>
      <div>{$value['title']}</div>
      <items-list></items-list>
   </li>
</ul>
Шаблонные ссылки

Если вам нужна только рекурсия, но не нужны методы, можно реализовать данный пример без создания компонента. Autosync предлагает для этого специальный атрибут asref: 


<div asmodel="data">
   <h1>Shopping list</h1>       
   <ul asmodel=".items" asref="items-list">
      <li>
         <div>{$value['title']}</div>
         <items-list></items-list>
      </li>
   </ul>
</div>

Встретив данный атрибут, Autosync автоматически создаст компонент с шаблоном, соответствующим значению outerHTML данного элемента.

Если данные приходят с сервера в формате с "лишним" корневым элементом, но вы не хотите делать трансформацию при получении, вы можете указать во внешнем шаблоне путь к нулевому элементу data.items.0 в атрибуте asmodel, поскольку целочисленные индексы являются абсолютно корректными ключами для Autosync.
