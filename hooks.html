Хуки

Хуки позволяют вам выполнять собственную логику перед обновлением DOM и после него. Например, вы можете выполнить трансформацию модели, добавив или удалив часть полей, перед обновлением DOM, и добавить нужные CSS классы определённым элементам и прикрепить необходимые JS обработчики сразу после обновления.


function removeExtraInfo() {
   data.users.forEach(function(user) {
      delete user.extra
   })
}         

function setHandlers() {
   // Setting JS handlers for everything
}

as.setBefore('data', removeExtraInfo)
as.setAfter('data', setHandlers)
as.add('data')

В Autosync есть два вида хуков: хуки модели и глобальные хуки. Первые срабатывают только при изменении той модели, к которой они привязаны, тогда как вторые - при изменении любой существующей модели. Для регистрации глобального хука нужно передать в качестве имени модели пустую строку.

Любой хук получает на вход два аргумента: в первом передаётся новое значение модели после обновления, а во втором - объект с подробной информацией об обновлении (контекст, ключ, логический путь к источнику события внутри модели и ряд других данных).
